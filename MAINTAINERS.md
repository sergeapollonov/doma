# Maintainers

## Primary Maintainer

- `@sergeapollonov`

## Maintainer Responsibilities

Maintainers are responsible for:

- reviewing pull requests;
- triaging issues;
- keeping the roadmap focused;
- protecting the privacy-first product direction;
- reviewing security reports;
- keeping documentation aligned with implementation;
- cutting releases when public milestones are reached.

## Review Expectations

Pull requests should be reviewed for:

- TypeScript correctness;
- product scope;
- privacy impact;
- UI consistency;
- localization coverage;
- accessibility;
- documentation updates.

## Project Direction

Doma should stay a calm family planning app and a useful open-source Expo reference project. Maintainers should avoid accepting changes that turn it into a generic project management tool, social network, finance app, or broad productivity platform.
