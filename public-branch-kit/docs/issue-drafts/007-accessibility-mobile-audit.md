# Accessibility: audit core mobile screens

Labels: `accessibility`, `design-system`

## Summary

Audit the core Doma screens for basic mobile accessibility.

## Context

Doma should be simple and quick to use. Accessibility supports that goal, especially for touch targets, text contrast, labels, and predictable navigation.

## Suggested Screens

- Welcome
- Login
- Create Family
- Today
- Calendar
- Tasks
- Shopping
- New Event

## Acceptance Criteria

- Important buttons have accessible labels.
- Touch targets meet the minimum size from design tokens.
- Text contrast is checked against the design system colors.
- Icon-only actions have labels or clear surrounding text.
- Findings are documented in a follow-up issue or PR.
