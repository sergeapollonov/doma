# Changelog

All notable changes to Doma will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/), and this project follows semantic versioning once public releases stabilize.

## [Unreleased]

### Added

- Agent guidance for Codex and AI-assisted contributors in `AGENTS.md`.

## [v0.1.0-alpha] - 2026-06-08

### Added

- Initial public alpha baseline for Doma.
- Expo, React Native, and TypeScript scaffold.
- Static app prototype with local mock data.
- Doma design tokens in `src/theme`.
- Russian and Polish copy structure in `src/i18n`.
- Product, design, user flow, data model, privacy, and OSS roadmap documentation.
- Visual design references and icon concepts.
- Public README, MIT license, contribution guide, code of conduct, security policy, and maintainer notes.
- GitHub Actions CI for TypeScript checks.
- Dependabot configuration.
- GitHub issue templates and pull request template.
- Public issue drafts for roadmap, localization, security, accessibility, testing, and design system work.

### Security

- Published as a sanitized public branch with no private chat history, local archives, credentials, or trash files.
- Added privacy-first contribution and security guidance.

### Known Limitations

- Real authentication is not implemented.
- Persistent local state is not implemented.
- Supabase backend sync is not implemented.
- Push notifications and widgets are not implemented.
- This release is not production-ready.
